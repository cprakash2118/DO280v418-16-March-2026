    Token based via OAuth:-
	 - OAuth is an Identity Broker.
	 - Oauth sync with Identity Provider (IdP).
	 What is IdP ?
	 - htpasswd (Local IdP) - Which keep the user's database within OCP cluster.
	 - LDAP 
	 - Keystone 
	 - OIDC (Open ID COnnect)
	 - Active DIrectory (AD).
	
	user1-5/redhat 
	
	Step-1-# Create Database (User's DB - user & Password).
	  $ htpasswd --help 
	  $ htpasswd -c -B -b htpasswd-file-name user1 redhat
	  $ htpasswd    -B -b htpasswd-file-name user2 redhat
	  $ htpasswd    -B -b htpasswd-file-name user3 redhat	  
      $ cat htpasswd-file-name
	  
	Step-2-# Place the created database to OCP cluster (Create secret).
	  $ oc create secret generic secret-htpasswd-a-team --from-file htpasswd=htpasswd-file-name -n openshift-config 
	  $ oc get secret secret-htpasswd-a-team -n openshift-config
	  $ oc get secret secret-htpasswd-a-team -n openshift-config -o yaml
	  
	Step-03-# UPdate to OAuth service.
	  $ oc get oauth 
	  $ oc get oauth -o yaml 
	  $ oc edit oauth 
	  or
	  $ oc get oauth -o yaml > oauth.yaml 
	  $ vim oauth.yaml 
	  $ oc apply -f oauth.yaml
